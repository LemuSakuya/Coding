# 第 7 章 漏洞评估与扫描

## 7.1 学习目标

1. 区分 **漏洞评估（VA）** 与 **渗透测试（PT）** 的边界、目标与产出。
2. 熟练使用主流扫描器（Nmap / Nessus / OpenVAS / Nuclei）完成资产与漏洞识别。
3. 理解 **CVE / CVSS / CWE / CPE** 四大标识符的关系。
4. 具备对扫描结果去噪、分级、闭环整改的能力。

---

## 7.2 关键术语

| 术语 | 含义 | 维护者 |
|------|------|--------|
| **CVE** | Common Vulnerabilities and Exposures，漏洞唯一编号 | MITRE |
| **CWE** | Common Weakness Enumeration，漏洞分类体系（如 CWE-79 XSS） | MITRE |
| **CPE** | Common Platform Enumeration，软件/硬件的标准标识 | NIST |
| **CVSS** | Common Vulnerability Scoring System，漏洞评分 0–10 | FIRST |
| **EPSS** | Exploit Prediction Scoring System，30 天内被利用概率 | FIRST |
| **KEV** | Known Exploited Vulnerabilities，CISA 实锤被利用清单 | CISA |

### 7.2.1 CVSS v3.1 评分

```
Base  ( AV:N / AC:L / PR:N / UI:N / S:U / C:H / I:H / A:H ) = 9.8 严重
Temporal  ( 漏洞利用是否公开、补丁状态 )
Environmental  ( 企业特定资产重要性 )
```

- 严重（Critical）：9.0–10.0
- 高（High）：7.0–8.9
- 中（Medium）：4.0–6.9
- 低（Low）：0.1–3.9

## 7.3 漏洞扫描的层次

```
┌───────────────────────────────────────┐
│  网络层：端口 / 服务 / 协议漏洞         │ ← Nmap, Masscan, Naabu
├───────────────────────────────────────┤
│  主机层：OS / 补丁 / 配置 / 弱口令      │ ← Nessus, OpenVAS, Lynis, WindowsAudit
├───────────────────────────────────────┤
│  应用层：Web / API / 业务逻辑          │ ← Burp, ZAP, Nuclei, Acunetix
├───────────────────────────────────────┤
│  代码层：SAST / SCA / DAST / IAST      │ ← SonarQube, Semgrep, Snyk, CodeQL
├───────────────────────────────────────┤
│  云层 / 容器：IAM / K8s / 镜像        │ ← Prowler, Trivy, kube-bench
└───────────────────────────────────────┘
```

## 7.4 Nmap 精讲

### 7.4.1 常用选项

```bash
# 存活发现
nmap -sn 192.168.1.0/24                       # ICMP/TCP SYN ping
nmap -Pn -p 80,443 target.com                 # 禁 ping，直接扫端口

# 端口扫描
nmap -sS -p- --min-rate=3000 target.com       # SYN 扫全端口
nmap -sU --top-ports 50 target.com            # UDP Top 50
nmap -sT -p 1-1024 target.com                 # 完整三次握手（无权限时）

# 服务识别 / 操作系统识别 / 脚本
nmap -sC -sV -O -A target.com

# 速度模板：T0（paranoid）- T5（insane）
nmap -T4 target.com

# 输出
nmap -oA scan_result target.com               # 同时输出 .nmap .gnmap .xml
```

### 7.4.2 NSE 脚本（按场景）

```bash
# 漏洞类
nmap --script vuln target.com
nmap --script smb-vuln-ms17-010 -p445 target.com

# 枚举类
nmap --script http-enum,http-title,http-headers -p80,443 target.com
nmap --script ssl-enum-ciphers -p443 target.com

# 暴破类（谨慎）
nmap --script ftp-brute --script-args userdb=users.txt,passdb=pass.txt -p21 target.com
```

### 7.4.3 Masscan + Nmap 组合

```bash
# 快速扫 65535 全端口，再让 Nmap 做指纹
sudo masscan -p1-65535 10.0.0.0/16 --rate=10000 -oL open.masscan
awk '/open/ {print $4":"$3}' open.masscan | sort -u > alive.txt
nmap -sV -sC -iL alive.txt -oA deep
```

## 7.5 Web 漏洞扫描器

### 7.5.1 Nuclei（模板化、现代化）

```bash
# 常用
nuclei -u https://target.com -severity critical,high
nuclei -l live.txt -t cves/ -t exposures/ -j -o result.json

# 自定义模板（示例：检测默认登录）
cat > tpl.yaml <<'EOF'
id: default-login-admin
info:
  name: Tomcat default login
  severity: high
http:
  - method: GET
    path:
      - "{{BaseURL}}/manager/html"
    matchers-condition: and
    matchers:
      - type: status
        status: [401]
      - type: word
        words:
          - 'Tomcat'
EOF
nuclei -u https://target.com -t tpl.yaml
```

### 7.5.2 Burp Suite 主动扫描

- **Community 版**：手动重放、代理、爬虫，无主动扫描。
- **Pro 版**：`Scanner` → `Crawl + Audit` → 使用 `Scan configuration` 自定义强度。
- 常用策略：`Audit checks - lightweight` （快）/ `Audit checks - extensive` （深）。

### 7.5.3 OWASP ZAP

```bash
# 快速全站扫
zap-cli quick-scan --spider -r https://target.com
# Docker 版（推荐）
docker run -v $(pwd):/zap/wrk/:rw -t owasp/zap2docker-stable \
  zap-baseline.py -t https://target.com -r zap_report.html
```

## 7.6 主机与基线扫描

### 7.6.1 Nessus / OpenVAS

- **Nessus Essentials**：免费，限 16 IP；适合个人与小环境。
- **OpenVAS (GVM)**：开源，含 6 万 + NVT；Docker 镜像 `greenbone/openvas`。
- 扫描模板：`Basic Network Scan` / `Advanced Scan` / `Credentialed Scan`（带凭证，准确度更高）。

### 7.6.2 Linux 基线（CIS Benchmark）

```bash
# Lynis：快速审计
sudo apt install lynis
sudo lynis audit system
sudo lynis audit system --quick --tests-from-category "authentication,filesystems"

# CIS-CAT Pro / OpenSCAP
sudo apt install scap-security-guide openscap-scanner
oscap xccdf eval --profile xccdf_org.ssgproject.content_profile_cis \
  --results-arf arf.xml --report report.html \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml
```

### 7.6.3 Windows 审计

- `SecPol.msc` / `gpedit.msc` 对照 CIS Windows Benchmark。
- Seatbelt（BloodHoundAD 系列工具）：`Seatbelt.exe -group=all`。
- `PingCastle`：企业域环境快速评分。

## 7.7 SAST / SCA / DAST

| 类别 | 含义 | 工具 |
|------|------|------|
| SAST | 静态源码分析 | SonarQube · Semgrep · CodeQL · Checkmarx |
| SCA | 软件组成分析（依赖漏洞） | Snyk · OSV-Scanner · Dependency-Check · Trivy |
| DAST | 运行时黑盒扫描 | ZAP · Burp · Acunetix |
| IAST | 运行时插桩白盒 | Contrast · Seeker |

### 7.7.1 Semgrep 快速上手

```bash
pipx install semgrep
semgrep --config=auto ./my-project
# 或者指定规则
semgrep --config=p/owasp-top-ten --config=p/security-audit ./src
```

### 7.7.2 Trivy（容器 / 依赖 / IaC 全能）

```bash
trivy fs --security-checks vuln,secret,config ./repo
trivy image nginx:1.24.0
trivy k8s --report summary cluster
```

## 7.8 扫描结果处理

### 7.8.1 去噪与分级

1. **过滤误报**：手工复现、带凭证复扫、交叉多扫描器。
2. **分级**：CVSS × 资产重要性 × 暴露面（对外网 > 内网）。
3. **可利用性**：参考 EPSS、KEV、Exploit-DB、GitHub PoC。
4. **修复优先级**：KEV + EPSS > 0.5 + 互联网暴露 → 立即修。

### 7.8.2 报告模板（核心字段）

```
1. 摘要：扫描范围、时间、工具、高危数量
2. 漏洞列表（按 CVSS 排序）：
   - 漏洞名 + CVE/CWE
   - 受影响资产
   - 严重性评分
   - 复现步骤 / 证据截图
   - 影响分析
   - 修复建议（短期 workaround + 长期 patch）
3. 基线不合规项
4. 整改跟踪表（责任人、期限、状态）
```

### 7.8.3 开源情报关联

```bash
# 拿到 CVE 后，一次性查可用 PoC
curl -s "https://cve.circl.lu/api/cve/CVE-2024-3400" | jq .
# searchsploit
searchsploit apache 2.4
# GitHub 搜索 PoC
curl -s "https://api.github.com/search/repositories?q=CVE-2024-3400" | jq '.items[].html_url'
```

## 7.9 实战：一次完整的漏洞评估流程

```
1. 授权：SOW 签字、资产白名单、时间窗口
2. 资产梳理：合并 CMDB + 被动侦察（见 Ch06）
3. 分层扫描：网络层 → 主机层 → Web 层
4. 凭证扫描：补做带 SSH / SMB 凭证的深度扫
5. 结果聚合：Excel / DefectDojo 导入去重
6. 人工复现：对高危样本抽样复现
7. 风险排序：KEV + EPSS + CVSS + 业务关键度
8. 整改方案：补丁 / 缓解 / 虚拟补丁（WAF rule）
9. 复扫验证：闭环前必须重新扫一次
10. 报告归档：版本化保存，纳入季度趋势分析
```

## 7.10 练习题

1. 在自己的 Kali 靶机网段用 Nmap + NSE `vuln` 脚本做一次完整扫描，手工核对至少 3 个高危项。
2. 在 GitHub 找一个开源 Web 项目，用 Semgrep + Trivy + ZAP 三合一扫描，输出一份 Markdown 报告。
3. 把最近一周的 CISA KEV 列表与公司资产的 CPE 做一次 "漏洞命中" 匹配练习。
4. 写一个 Python 脚本：读取 Nessus 的 `.nessus` XML，按 CVSS ≥ 7 + 存在于 KEV 的规则过滤，输出整改清单。

## 7.11 小结

扫描只是手段，**风险语言**才是目的：你要把一堆 CVE 翻译成业务能听懂的 "会不会被黑、会丢什么、什么时候修"。
