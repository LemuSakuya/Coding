# 第 1 章 网络安全绪论与学习路径

## 1.1 学习目标

1. 理解网络安全的核心使命：保护信息系统的 **机密性 / 完整性 / 可用性**（CIA 三元组）。
2. 掌握安全领域常用术语与威胁分类。
3. 建立"攻—防—管"三位一体的知识地图。
4. 了解白帽子的法律与伦理边界。

---

## 1.2 核心概念

### 1.2.1 CIA 三元组

| 属性 | 含义 | 对应威胁 | 典型技术 |
|------|------|----------|----------|
| Confidentiality 机密性 | 信息只对授权者可见 | 窃听、泄露 | 加密、访问控制、脱敏 |
| Integrity 完整性 | 信息未被未授权篡改 | 篡改、伪造 | 哈希、MAC、数字签名 |
| Availability 可用性 | 授权者可正常访问 | DoS/DDoS、勒索 | 冗余、限流、备份、WAF |

扩展为 **Parkerian 六元组**：CIA + 真实性（Authenticity）、占有性（Possession）、效用性（Utility）。

### 1.2.2 STRIDE 威胁模型

| 字母 | 威胁 | 破坏的属性 |
|------|------|-----------|
| S | Spoofing 冒充身份 | Authentication |
| T | Tampering 篡改 | Integrity |
| R | Repudiation 抵赖 | Non-repudiation |
| I | Information Disclosure 信息泄露 | Confidentiality |
| D | Denial of Service 拒绝服务 | Availability |
| E | Elevation of Privilege 权限提升 | Authorization |

### 1.2.3 风险公式

```
风险 Risk = 威胁 Threat × 漏洞 Vulnerability × 资产 Asset
                         ────────────────────
                              安全措施 Control
```

风险处置四策略：**接受 / 转移 / 规避 / 缓解**。

### 1.2.4 防御深度（Defense in Depth）

```
物理安全 → 网络边界 → 主机 → 应用 → 数据 → 用户教育
```

每一层都不可单独依赖，必须叠加布防。

## 1.4 攻击链（Cyber Kill Chain）

Lockheed Martin 提出的 7 阶段模型：

1. **Reconnaissance** 侦察 —— OSINT、端口扫描
2. **Weaponization** 武器化 —— 制作 payload（如带 Macro 的 Word）
3. **Delivery** 投递 —— 钓鱼邮件、水坑、U 盘丢包
4. **Exploitation** 利用 —— 触发漏洞执行 shellcode
5. **Installation** 植入 —— 安装后门 / RAT
6. **Command & Control** 外联 —— C2 通信
7. **Actions on Objectives** 达成目的 —— 数据窃取、破坏

### MITRE ATT&CK 框架

比 Kill Chain 更细粒度，按 **Tactics（战术）→ Techniques（技术）→ Procedures（过程）** 拆分：

- 14 类战术：TA0001 Initial Access → TA0040 Impact
- 查询入口：<https://attack.mitre.org/matrices/enterprise/>

## 1.5 学习路径建议

### 新手三问

1. **我的目标是？** CTF？企业蓝队？甲方安全？乙方渗透？
2. **我的基础是？** 编程几年？网络协议熟不熟？是否懂操作系统内核？
3. **我的时间预算？** 每天 1h vs 3h，半年可达到的高度完全不同。

### 典型路径

| 目标 | 推荐顺序 |
|------|----------|
| **Web 渗透** | 网络协议 → Linux → Python → HTTP → OWASP Top 10 → Burp → CTF Web |
| **内网渗透** | Windows/AD → PowerShell → Mimikatz → BloodHound → C2 框架 |
| **Pwn / 逆向** | C / 汇编 → gdb → 栈溢出 → ROP → 堆利用 → 内核 pwn |
| **蓝队 / SOC** | 网络协议 → 日志分析 → ELK/Splunk → ATT&CK → Sigma 规则 |
| **恶意代码分析** | PE/ELF → 汇编 → IDA/Ghidra → Sandbox → YARA |
| **云原生安全** | Docker → K8s → IAM → Trivy/Falco → eBPF |

## 1.6 白帽子的伦理与法律

### 中国大陆相关法律

| 法律 / 法规 | 关键条款 |
|------------|---------|
| 《刑法》第 285 条 | 非法侵入计算机系统罪 |
| 《刑法》第 286 条 | 破坏计算机信息系统罪 |
| 《刑法》第 287 条之二 | 帮助信息网络犯罪活动罪 |
| 《网络安全法》| 关键信息基础设施保护、数据出境、通报机制 |
| 《数据安全法》| 重要数据分类分级 |
| 《个人信息保护法》| 个人信息处理合法性 |
| 等保 2.0 GB/T 22239-2019 | 分 5 级定级保护 |

### 红线

- **禁止未授权扫描 / 渗透** 任何不属于你、未取得书面授权的目标。
- **禁止在公网散播** 0day、恶意样本。
- **禁止利用漏洞牟利、勒索**。
- SRC 上报漏洞请走官方渠道，遵守"负责任披露"。
