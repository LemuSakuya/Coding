## 4.3 对称加密

### 4.3.1 分组密码 vs 流密码

| 类型 | 代表 | 特点 |
|------|------|------|
| 分组密码 | AES-128/256、SM4 | 固定块长（128 bit） |
| 流密码 | ChaCha20、RC4（已废） | 生成密钥流 XOR |

### 4.3.2 工作模式

| 模式 | 描述 | 安全性 |
|------|------|--------|
| ECB | 每块独立加密 | **不安全**，明显 pattern（企鹅图实验） |
| CBC | 上一块密文 XOR 下一块明文 | 需要 IV；Padding Oracle 风险 |
| CTR | 计数器 + 密钥流 | 并行，IV 不能重用 |
| GCM | CTR + GHASH 认证 | **推荐**，AEAD |
| CCM | CBC-MAC + CTR | 轻量，常用于 WiFi WPA2 |
| XTS | 磁盘加密专用 | BitLocker、LUKS |

### 4.3.3 常见误用

- **ECB 加密头像** → 泄露结构
- **CBC + 固定 IV** → 相同明文 = 相同密文
- **GCM 重用 nonce** → 灾难级，攻击者可恢复认证密钥
- **Padding Oracle**（POODLE、Lucky13）→ 逐字节解密

### 4.3.4 Python 示例

```python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

key = AESGCM.generate_key(bit_length=256)
aesgcm = AESGCM(key)
nonce = os.urandom(12)              # GCM 推荐 96 bit
ct = aesgcm.encrypt(nonce, b"plain", b"aad")
pt = aesgcm.decrypt(nonce, ct, b"aad")
```

## 4.4 非对称加密

### 4.4.1 RSA

- 基于大整数分解困难
- 密钥长度推荐 ≥ 3072 bit（相当于 128-bit 安全）
- **必须使用 OAEP 填充**，PKCS#1 v1.5 已不安全

### 4.4.2 椭圆曲线（ECC）

- 相同安全强度下密钥更短：**256-bit ECC ≈ 3072-bit RSA**
- 常用曲线：**Curve25519 / Ed25519 / P-256（secp256r1）**
- 签名：ECDSA / EdDSA

### 4.4.3 Diffie-Hellman 密钥交换

```
公共参数：大素数 p、生成元 g
Alice: a 随机, A = g^a mod p
Bob:   b 随机, B = g^b mod p
Alice 收到 B → 计算 K = B^a mod p
Bob 收到 A   → 计算 K = A^b mod p
```

**中间人攻击**（MitM）：需要认证 → TLS 证书、SSH 指纹。

**前向安全**：一次性临时密钥（DHE / ECDHE）→ 私钥泄露也不能解过去的流量。

### 4.4.4 常见 CTF 考点

- **低加密指数**（e=3）+ 小明文 → 直接开三次方
- **共模攻击** / **广播攻击**
- **Wiener 攻击**：d 过小
- **Coppersmith**：已知部分 p 或 部分明文
- **Pollard's Rho**：小素数分解

Python 工具：`gmpy2` + `sympy` + `pycryptodome` + `PwnTools` 的 `sage`。

## 4.5 哈希函数

### 4.5.1 性质

- **抗原像**：给定 h(x)，难以找 x
- **抗第二原像**：给定 x，难以找 x' 使 h(x')=h(x)
- **抗碰撞**：难以找两个 x ≠ y 使 h(x)=h(y)

### 4.5.2 主流算法

| 算法 | 输出 | 状态 |
|------|------|------|
| MD5 | 128 bit | **碰撞已破** |
| SHA-1 | 160 bit | **碰撞已破**（SHAttered） |
| SHA-256 / 512 | 256 / 512 bit | 推荐 |
| SHA-3（Keccak） | 可变 | 推荐，海绵结构 |
| BLAKE2 / BLAKE3 | 可变 | 高性能 |
| SM3 | 256 bit | 国密 |

### 4.5.3 口令哈希（慢哈希）

```
bcrypt / scrypt / Argon2  ← 推荐（加盐 + 迭代 + 内存硬）
PBKDF2-HMAC-SHA256        ← FIPS 合规但易被 GPU 加速
MD5 / SHA1 / SHA256       ← 裸 hash 用于口令 = 错误
```

口令存储最佳实践：`Argon2id(password, salt, m=64MB, t=3, p=4)`。

## 4.6 MAC 与数字签名

| 机制 | 密钥 | 提供 |
|------|------|------|
| MAC（HMAC） | 对称 | 完整性 + 源认证 |
| 数字签名（RSA/ECDSA） | 非对称 | 完整性 + 源认证 + **不可否认** |

### 4.6.1 HMAC 构造

```
HMAC(K, m) = H( (K ⊕ opad) || H( (K ⊕ ipad) || m ) )
```

### 4.6.2 签名流程

```
发送方：  sig = Sign(priv, H(m))
接收方：  Verify(pub, H(m), sig)
```

### 4.6.3 常见误用

- 用 `==` 比较 MAC / signature → **时序攻击** → 应使用 `hmac.compare_digest`
- **Hash Length Extension**（MD5/SHA1/SHA256）：`H(secret || m)` 可被扩展。防御：改用 HMAC。

## 4.7 PKI 与证书

```
  根 CA (离线) ──签发──▶ 中间 CA ──签发──▶ 服务器证书
                                              │
                                     浏览器/OS 信任根 CA
```

- **X.509** 证书字段：Subject、Issuer、Public Key、SAN、Extensions
- 吊销：**CRL**（列表）/ **OCSP**（在线查询）/ **OCSP Stapling**
- **CAA DNS 记录**：限制哪家 CA 可签发本域证书
- **CT（Certificate Transparency）**：强制公开所有证书

## 4.8 现代协议组合

### Signal 端到端加密（简化）

```
身份密钥 IK + 签名预共享密钥 SPK + 一次性预共享密钥 OPK
→ X3DH 协商根密钥
→ Double Ratchet 每条消息演进
→ 前向安全 + 后向安全
```

### TLS 1.3 密码套件

```
TLS_AES_128_GCM_SHA256
TLS_AES_256_GCM_SHA384
TLS_CHACHA20_POLY1305_SHA256
```

### JWT 常见坑

- `alg: none` 绕过
- RS256 → HS256 混淆（用 RSA 公钥当 HMAC 密钥）
- kid 注入、JKU 外连

## 4.9 后量子密码（PQC）

Shor 算法对整数分解 / 离散对数多项式时间 → RSA/ECC 可被量子计算机击碎。

NIST PQC 选中算法（2024）：

| 类型 | 算法 |
|------|------|
| KEM（密钥封装） | **ML-KEM**（原 Kyber） |
| 签名 | **ML-DSA**（Dilithium）、**SLH-DSA**（SPHINCS+）、**FALCON** |

工程上：Chrome / Cloudflare 已开始混合部署 X25519+Kyber768。
