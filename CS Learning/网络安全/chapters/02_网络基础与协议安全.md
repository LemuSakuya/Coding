## 2.2 TCP/IP 协议栈

```
┌──────────────────────────────────────────────┐
│ 应用层  HTTP(S) / DNS / FTP / SSH / SMTP     │
├──────────────────────────────────────────────┤
│ 传输层  TCP / UDP / QUIC                     │
├──────────────────────────────────────────────┤
│ 网络层  IPv4 / IPv6 / ICMP / IPSec          │
├──────────────────────────────────────────────┤
│ 链路层  Ethernet / ARP / WiFi 802.11        │
├──────────────────────────────────────────────┤
│ 物理层  光 / 铜 / 无线电                      │
└──────────────────────────────────────────────┘
```

### 2.2.1 TCP 三次握手 / 四次挥手

```
Client                                        Server
  │ ── SYN (seq=x) ───────────────────────────▶ │
  │ ◀─ SYN+ACK (seq=y, ack=x+1) ─────────────── │
  │ ── ACK (ack=y+1) ────────────────────────▶ │
  │               已建立 (ESTABLISHED)           │
  │ ...                                          │
  │ ── FIN ──────────────────────────────────▶ │
  │ ◀─ ACK ────────────────────────────────────│
  │ ◀─ FIN ────────────────────────────────────│
  │ ── ACK ──────────────────────────────────▶ │
  │              TIME_WAIT (2 MSL)               │
```

**攻击面**：
- **SYN Flood**：伪造源 IP 疯狂 SYN，耗尽半连接队列。
- **Reset Attack**：猜测 seq/ack 发送 RST 终止连接。

### 2.2.2 UDP 与 QUIC

- UDP 无连接、无状态 → DNS、DHCP、NTP、SNMP、视频流
- **UDP 反射放大攻击**：DNS / NTP / Memcached 放大比可达 1:50000

## 2.3 IP & ICMP

### IPv4 头部关键字段

| 字段 | 安全关注点 |
|------|-----------|
| TTL | 指纹识别（Linux 64 / Win 128 / 部分设备 255）|
| 标志 DF / MF | 分片攻击（Ping of Death、Teardrop） |
| 协议号 | 1=ICMP / 6=TCP / 17=UDP / 50=ESP |
| 源 IP | 易被伪造 → uRPF、BCP38 防范 |

### ICMP 滥用

- `ping` 探测 → 可封禁
- **ICMP 隧道**：把 C2 数据塞进 echo request 载荷（工具：`icmptunnel`、`ptunnel`）

## 2.4 ARP 与链路层攻击

### 2.4.1 ARP 欺骗原理

```
受害者 PC   ──── 广播 "谁有 192.168.1.1?" ───▶
攻击者      ◀── 抢答 "我是 1.1，我的 MAC 是 AA:..." ──
受害者 PC   → 发送所有流量到攻击者
```

**工具**：`arpspoof`（dsniff 套件）、`ettercap`、`bettercap`。

### 2.4.2 防御

- 静态 ARP 绑定
- 交换机 **DAI**（Dynamic ARP Inspection）+ DHCP Snooping
- 端口绑定 MAC

### 2.4.3 其他链路层攻击

- **MAC 洪泛**：填满 CAM 表 → 交换机退化成 HUB
- **VLAN 跳跃**：Double Tagging / Switch Spoofing
- **STP BPDU 攻击**：伪造根桥

## 2.5 DNS 安全

### 2.5.1 解析流程

```
用户 → 本地 Resolver → 根 . → TLD .com → 权威 example.com
```

每一跳都是攻击面。

### 2.5.2 典型攻击

| 攻击 | 原理 | 防御 |
|------|------|------|
| DNS 劫持 | 运营商 / 恶意 Resolver 替换结果 | DoH / DoT、DNSSEC |
| Cache Poisoning | 伪造响应抢先到达 Resolver | 随机源端口 + DNSSEC |
| DNS 隧道 | 把数据编码进子域名 | 检测异常长度 / 频率 |
| DGA 域名 | 恶意软件周期性生成 C2 域名 | 威胁情报、机器学习 |
| 子域名接管 | CNAME 指向已失效云资源 | 清理孤儿记录 |

### 2.5.3 DoH / DoT

- **DoH**：DNS over HTTPS（443，难以被中间设备识别）
- **DoT**：DNS over TLS（853，易识别易封禁）

## 2.6 HTTP / HTTPS 安全要点

### 2.6.1 HTTP 版本

| 版本 | 传输 | 复用 | 关键安全点 |
|------|------|------|-----------|
| HTTP/1.1 | TCP | pipeline | Smuggling、Host 头注入 |
| HTTP/2 | TCP + TLS | 多路复用、头压缩 | HPACK 攻击、Rapid Reset CVE-2023-44487 |
| HTTP/3 | QUIC/UDP | 0-RTT | 0-RTT 重放风险 |

### 2.6.2 重要安全头

```http
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
Content-Security-Policy: default-src 'self'; script-src 'self' 'nonce-xxx'
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=(), geolocation=()
```

### 2.6.3 HTTP Smuggling

前端与后端对 `Content-Length` 与 `Transfer-Encoding` 解析不一致 → 请求走私。

变体：**CL.TE / TE.CL / TE.TE**。

## 2.7 TLS / SSL

### 2.7.1 TLS 1.3 握手（1-RTT）

```
Client                                       Server
  │ ClientHello (支持的套件、key_share) ───────▶ │
  │ ◀── ServerHello + EncryptedExtensions ────  │
  │      + Certificate + CertVerify + Finished  │
  │ ── Finished (Application Data) ───────────▶ │
```

**对比 TLS 1.2**：
- 去除 RSA 密钥交换，只保留 (EC)DHE → 天然 **前向安全**
- 禁用 CBC、RC4、SHA1、压缩、重协商

### 2.7.2 经典漏洞

| 漏洞 | 年份 | 危害 |
|------|------|------|
| Heartbleed (CVE-2014-0160) | 2014 | 内存泄露读取私钥 |
| POODLE | 2014 | SSL 3.0 CBC 填充 oracle |
| FREAK / Logjam | 2015 | 降级到弱 DH/RSA |
| ROBOT | 2017 | RSA PKCS#1 v1.5 oracle |
| BEAST / CRIME / BREACH | 2011-2013 | CBC / 压缩攻击 |

### 2.7.3 证书链与 PKI

```
根 CA → 中间 CA → 服务器证书
     ↓              ↓
   自签名        OCSP / CRL 吊销
```

- **证书透明度 CT**：所有证书需公开到 CT Log（Google / Cloudflare）
- **HPKP** 已弃用，改用 CT + Expect-CT / CAA 记录

## 2.9 工具速查

| 工具 | 用途 | 典型命令 |
|------|------|----------|
